# MVP Scope

## Goal

Prove that the platform can support a community-scale survival experience with persistent state, visible player creativity, event-driven progression, and operator-friendly tooling.

## Included

- one playable world zone
- one road or travel corridor
- one settlement loop with placeable structures
- one inventory pickup and storage loop
- one guild model
- one rank or progression ladder
- one admin-defined challenge or event
- Discord identity link and one role sync path
- distant settlement signatures or outlines
- region partition and interest management basics
- operator UI for player, guild, and event inspection

## Excluded

- full combat roster
- full biome system
- all weather and environment simulation
- complex economy or market systems
- full crafting tree
- PvP balancing
- mobile or console support

## Done Means

The MVP is done when:
- players can connect to the same region
- movement and placement are authoritative
- a placed structure persists across restart
- a guild challenge can be configured and completed
- the resulting event trail is visible in the admin UI
- Discord role or announcement integration reflects the outcome
- distant settlements are visible before their full structure details load

## Exit Criteria For MVP

- trusted-alpha group can complete a guided session
- operators can answer why progress changed
- server metrics show stable relevance partitioning under a moderate test load
